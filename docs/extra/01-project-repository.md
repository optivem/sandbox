# Project Repository

## 1. Define Project Member(s)

**Case 1: Individual Plan**

If you have an Individual Plan:

- Project Member(s): Yourself

**Case 2: Team Plan**

If your Engineering Leader had arranged for you and your colleagues to get a Team Plan, and work together as a team, then:

- Project Member(s): Employees who received access. You'll work together on this project as a team, rather than working individually. This will help you practice teamwork, support each other, and move along faster.

In that case, you should do the activities below together.

## 2. Choose System Domain

Write down (on paper) the domain for your Sandbox Project.

Feel free to pick whatever domain you want.

- Book Store
- Task Planner
- TODO List
- Flight Reservation System
- Anything else...

*Note: I'd recommend that you do NOT choose to make a generic eShop. The reason is because I'll be making an eShop example throughout the series, and then if you build an eShop as well, most likely you'll just copy my solutions and you won't learn much. That's why, could you please choose something that is NOT a generic eShop.*

**NDA Compliance**

Please make sure that you don't violate your company's NDA. To ensure NDA compliance, these are some minimal requirements:

1. Please do NOT choose a domain that your company works in. For example, if your company makes logistics software, then please do NOT make a Sandbox Project in the logistics domain. The primary reason is, to avoid leakage of business logic. That's why you should pick a completely different domain. Avoid any domain logic that's related to your project at work.
2. Please do NOT share your company data in the project.

As you build your Sandbox Project, everything will be public & open source.

## 3. Choose System Name

Choose a name for your system.

*Example: ACME Shop.*

## 4. Create System Repository

Create a System Repository in GitHub. This will be the entry point for your project on GitHub.

1. Repository Name: Hyphenate your System Name, with lowercase letters (e.g. for the ACME Shop System, we call the repository acme-shop).
2. Choose Visibility: Public
3. Add Readme: On
4. Add License: MIT License (or another Open Source license)

> *Note: If you're working together as a team, please make ONE System Repository for the whole team, rather than making individual repositories. In this way, you'll practice teamwork.*

## 5. Add Project Member(s) as Collaborators

If you're working by yourself, skip this step.

If you're working in a team, you need to add other members as collaborators.

In your GitHub Repository:

1. Go to Settings → Collaborators → Manage Access → Add people.
2. Search for username
3. Click "Add to repository"
